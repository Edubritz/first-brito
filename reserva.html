Sintaxe e Semântica

Sintaxe e Semântica

No contexto de programação, a sintaxe se refere às regras de estrutura da linguagem, ou seja, a forma correta de escrever os códigos. Ela define como as palavras e caracteres devem ser organizados para formar comandos válidos. Um erro sintático ocorre quando o código escrito foge das regras da linguagem, como quando, por exemplo, não é informado ponto e vírgula ao final de uma linha de um código em C.

Já a semântica se refere ao significado dos comandos e expressões dentro da linguagem. É o comportamento do código quando executado.

A estrutura básica de sintaxe do Erlang pode ser definida em módulos e funções. Um arquivo de código Erlang geralmente é um módulo, e cada função deve pertencer a um módulo. As funções são definidas por função/Aridade, onde aridade representa o número de parâmetros.

Palavras Reservadas

Palavras Reservadas

  • after: Usada no contexto de blocos receive para definir uma ação a ser realizada se nenhuma mensagem for recebida dentro de um determinado período de tempo.
  • and: Operador lógico E (AND). Avalia dois valores booleanos e retorna true apenas se ambos forem true.
  • andalso: Semelhante a and, mas é um operador de curto-circuito. Ele só avalia o segundo operando se o primeiro for true.
  • band: Operador lógico de bitwise AND. Aplica a operação bit a bit entre dois inteiros.
  • begin: Usada para iniciar um bloco de código que agrupa múltiplas expressões.
  • bnot: Operador bitwise NOT. Inverte todos os bits de um número inteiro.
  • bor: Operador bitwise OR. Realiza a operação bit a bit de OU entre dois inteiros.
  • bsl: Operador de shift à esquerda (bitwise shift left). Desloca os bits de um número inteiro para a esquerda.
  • bsr: Operador de shift à direita (bitwise shift right). Desloca os bits de um número inteiro para a direita.
  • bxor: Operador bitwise XOR. Aplica a operação XOR (OU exclusivo) bit a bit entre dois inteiros.
  • case: Usada para realizar pattern matching em expressões e executar diferentes blocos de código com base no valor correspondente.
  • catch: Usada para capturar exceções em Erlang.
  • div: Operador de divisão inteira.
  • else: Usada no contexto de expressões if para definir o bloco de código a ser executado se nenhuma das condições anteriores for satisfeita.
  • end: Usada para indicar o fim de blocos como if, case, begin.
  • fun: Define uma função anônima (lambda).
  • if: Estrutura de controle condicional para verificar condições booleanas.
  • not: Operador lógico de negação (NOT).
  • receive: Usada para esperar e receber mensagens em processos concorrentes.
  • rem: Usada para resto de divisão (módulo).
  • try: Usada para capturar exceções em Erlang.

Módulos e Funções

Módulos e Funções

Para executar programas com diferentes módulos e funções, é recomendado utilizar algum editor de texto para criar o arquivo do código. É importante que o arquivo criado esteja no mesmo diretório onde você começou o shell do Erlang e que também possua o mesmo nome do módulo informado no código, conforme o exemplo abaixo:

-module(tut).
-export([double/1]).

double(X) ->
    2 * X.

É possível rodar o programa pelo terminal através do seguinte comando no shell "c(tut).", e se compilar sem erro será impressa a seguinte mensagem: {ok, tut}. Caso tenha erro, a mensagem exibida será {error, tut}.

Erlang

Erlang

Erlang tem seu próprio shell onde é possível escrever códigos diretamente e verificar os resultados. Ao acessar o terminal e digitar erl, será apresentada a versão instalada. Neste momento, já é possível executar códigos simples, como operações matemáticas, como por exemplo “5 + 9.”. Em Erlang, cada expressão termina com um ponto final.

Tipo de Dados

Tipo de Dados

  • Inteiros: Inteiros são escritos como sequências de dígitos decimais, por exemplo, 12, 12375 e -23427 são inteiros.
  • Floats (Ponto Flutuante): Números de ponto flutuante são usados para representar valores com casas decimais.
  • Booleans: Erlang usa os átomos true e false para representar valores lógicos.
  • Binaries: Sequências compactas de bits usadas em dados binários, como arquivos.
  • Strings: Listas de inteiros onde cada número corresponde ao código ASCII de um caractere.
  • Átomos: Variáveis começam com letra maiúscula e átomos com letra minúscula.

  • Tuplas: As tuplas são utilizadas para agrupar coisas e torná-las mais compreensíveis. No Erlang, são indicadas por chaves: "{centimeter, 5}".
  • Listas: As listas em são cercadas por colchetes, como por exemplo: "[{moscow, {c, -10}}, {stockholm, {c, -4}}]".
  • Mapas: São um conjunto de associações de chave para valor. São encapsuladas com "#{ " e " }": "#{ "key" => 42 }".
  • Funções: Funções também são tipos de dados em Erlang. Você pode passar funções como argumentos, retorná-las de outras funções e armazená-las em variáveis.
  • Referências: São identificadores únicos, normalmente usados para rastrear processos ou transações. São criadas com a função make_ref().
  • Ports: São interfaces para comunicação entre Erlang e o mundo externo, como processos do sistema operacional ou dispositivos de hardware.
  • PIDs (Process Identifiers): São identificadores únicos para processos Erlang. Eles são usados para se referir a processos e enviar mensagens para eles.
  • Records (Registros): São estruturas de dados nomeadas que permitem agrupar dados relacionados. Eles são semelhantes a "structs" em C. Mas, tecnicamente, são uma conveniência sintática para tuplas.

Concorrência

Concorrência

Uma das principais razões para usar Erlang é pela capacidade de lidar com simultaneidade e programação distribuída. Por simultaneidade entende-se programas que podem lidar com vários threads de execução ao mesmo tempo.

Quando falamos de simultaneidade, podemos usar seu computador para várias tarefas como escrever um texto, mexer em uma planilha, ver e-mails e imprimir algo, tudo ao mesmo tempo. Embora o processador do seu computador faça uma coisa de cada vez, ele troca rapidamente entre essas tarefas, dando a impressão de que elas estão acontecendo simultaneamente.

Em Erlang, é fácil criar threads de execução paralelas, chamadas de processos, que funcionam ao mesmo tempo e podem se comunicar entre si. Isso mostra por que Erlang se destaca e é ideal para lidar com múltiplas tarefas simultâneas.
