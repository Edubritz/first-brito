Contexto Histórico

Erlang é uma linguagem de programação que foi criada por alguns engenheiros que trabalhavam para uma empresa de telefones chamada Ericsson. Eles queriam uma forma de criar programas que nunca parassem de funcionar, mesmo que algo desse errado, como quando estamos usando o celular e não queremos que a ligação caia. (Fonte)

Para explicar de forma simples, imagine que você tem várias pessoas em um parque de diversões, e cada uma está em uma fila diferente para um brinquedo. Cada pessoa tem sua própria fila e elas não precisam ficar brigando para usar a mesma fila. Se uma fila tiver problemas (como um brinquedo quebrado), as outras filas continuam funcionando normalmente. O Erlang funciona assim: ele permite que muitas partes de um programa trabalhem ao mesmo tempo, e se uma parte der problema, as outras continuam funcionando.

Ele foi criado para ajudar a manter as chamadas telefônicas funcionando sem interrupções, e hoje também é usado para aplicativos que precisam funcionar o tempo todo, como mensagens de texto ou bancos online.

Características

Aspectos que tornam a programação declarativa única e poderosa

Quando se fala em o que é Erlang logo se associa à OTP, que é um conjunto de bibliotecas Erlang. (Fonte)

Ele representa princípios de design com middleware para desenvolver sistemas. Além disso, possui um banco de dados distribuído, aplicativos para interface juntamente com outras linguagens, manipulação de lançamentos e ferramentas de depuração. (Fonte)

Em síntese, a linguagem criada pela Ericsson é funcional (se baseia em funções matemáticas), trabalha o conceito da imutabilidade e é tolerante a falhas.

Influências no Desenvolvimento do Erlang

Principais linguagens e conceitos que inspiraram a criação do Erlang

Influências

  • Prolog: A versão inicial do Erlang foi implementada em Prolog, uma linguagem de programação lógica. Essa escolha proporcionou à Erlang uma base sólida para o tratamento de padrões e a manipulação de dados de forma declarativa.
  • PLEX: Outra linguagem de programação, PLEX, utilizada em centrais telefônicas da Ericsson, também serviu de inspiração para o desenvolvimento do Erlang.
  • Conceitos de Concorrência e Distribuição: A Erlang foi projetada desde o início para suportar a concorrência e a distribuição de processos, características essenciais para sistemas telefônicos que precisam lidar com um grande número de chamadas simultâneas.

Características-chave do Erlang

Aspectos essenciais que tornam a Erlang poderosa e única

Características-chave

  • Concorrência: A Erlang permite a criação de milhares de processos leves, tornando-a ideal para aplicações altamente concorrentes. (Fonte)
  • Tolerância a Falhas: A linguagem possui mecanismos robustos para lidar com falhas de processos e garantir a continuidade do serviço. (Fonte)
  • Distribuição: A Erlang suporta a distribuição de processos em diferentes máquinas, facilitando a construção de sistemas escaláveis.
  • Alta Disponibilidade: A linguagem foi projetada para criar sistemas que permaneçam disponíveis continuamente, mesmo durante atualizações ou falhas.

Vantagens do Erlang

Principais benefícios que fazem do Erlang uma escolha poderosa

Vantagens

  • Concorrência Nativa: Erlang é projetado para lidar com milhares de processos simultâneos, facilitando a construção de sistemas altamente concorrentes.
  • Tolerância a Falhas: A linguagem suporta o conceito de "let it crash", permitindo que os sistemas se recuperem automaticamente de falhas sem comprometer a estabilidade geral.
  • Distribuição: Erlang facilita a construção de aplicações distribuídas, permitindo que processos se comuniquem facilmente entre diferentes nós.
  • Hot Code Swapping: É possível atualizar o código de uma aplicação em execução sem interrupção, o que é essencial para sistemas críticos que precisam de alta disponibilidade.
  • Imutabilidade: Os dados em Erlang são imutáveis, o que reduz a complexidade e os problemas relacionados a estados compartilhados.
  • Desempenho em Sistemas de Tempo Real: A capacidade de gerenciar muitos processos simultaneamente e a baixa latência tornam Erlang adequado para sistemas de tempo real.

Desvantagens do Erlang

Algumas das limitações que devem ser consideradas ao usar Erlang

Desvantagens

  • Curva de Aprendizado: Para desenvolvedores acostumados a linguagens imperativas, a transição para um modelo funcional e concorrente pode ser desafiadora.
  • Desempenho em Cálculos Intensivos: Embora Erlang seja excelente para sistemas concorrentes, ele pode não ser a melhor escolha para tarefas computacionais intensivas, onde linguagens como C ou Rust podem ter um desempenho superior.
  • Ecossistema Menor: Embora Erlang tenha uma comunidade ativa, seu ecossistema de bibliotecas e frameworks é menor comparado a linguagens como Python ou Java.
  • Tipagem Dinâmica: A tipagem dinâmica pode levar a erros que só são descobertos em tempo de execução, o que pode ser um problema em projetos grandes.
  • Menos Popularidade: A demanda por desenvolvedores de Erlang é menor em comparação com outras linguagens, o que pode dificultar a contratação de profissionais qualificados.
